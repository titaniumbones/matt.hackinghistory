Deploying Course Websites with git hooks.

Table of Contents Repository setup Workflow Now that I’ve moved my course websites to Hugo via ox-hugo, life is definitely easier than when I used WordPress, and my teaching is unquestionably crawling a little bit closer to actual reproducibility. It’s been non-trivial to figure out a good deployment process, though. My course materials live in a slightly broader context than my websites: I have my readings, my private notes, my grades, and a few other documents like teaching contracts, etc., in the main course directory. Some but not all of these materials can be committed to Git, so it makes intuitive sense to me to have a course repository which exists independently of the website. If I didn’t use ox-hugo to manage the org-to-hugo process – if, say, I used the somewhat more limited googrgeous support which is built into Hugo – I might try to build the website directly from the course materials. Instead, I’ve come up with a somewhat more convoluted process. It’s by no means perfect, and there are probably better solutions , but for now this process is working pretty well for me. Documenting here in case it’s useful to anyone else, and also in case I forget what I’ve done! For this post I’ll take my Digital History course as an example, but I’m hoping to move all my courses to the same system, semester by semester. Repository setup Each course has two repositories: a main repo, mostly for my own use, but potentially of use to colleagues. These sites are small, with one file per content type &#x2013; usually one each for syllabus, assignments, lectures, and some kind of &ldquo;other&rdquo; category. a website repo, containing one file per webpage &#x2013; so instead of a single Assignments.org, an entry for each individual assignment. These are markdown files created by org-hugo-export-subtree. The website repo has two relevant branches: a master branch that contains the uncompiled source files, and a gh-pages branch that holds the Hugo-generated files that are actually served on the server. The gh-pages branch is checked out as a git worktree as described in the Hugo docs. It took me a little while to figure this out, but once you understand the concept of git worktrees it&rsquo;s kind of fun to do things this way. For Digital History, I&rsquo;m using the excellent docdock theme, and as they suggest, I&rsquo;ve installed the theme as a git submodule. Workflow I do all my editing inside Emacs, in the original org-mode files that live in the &ldquo;main&rdquo; repo. Usually changes belong to one of two categories: &ldquo;features&rdquo; &#x2013; planned updates to assignments, course materials, due dates, etc. &ldquo;hotfixes&rdquo; &#x2013; corrections to errors, omissions, duplications in the course materials. I have to do these more often than I&rsquo;d like to admit! In either case, the workflow should look like this: make the change in source (.org) text export change to Hugo inspect changes on local server if changes look good, commit them to master in the main site commit those changes to the web master run hugo -s in the website master dir; commit changes to gh-pages branch; push gh-pages to upstream. There are a lot of steps there! Fortunately we can automate or semi-automate them. I&rsquo;ve left the first two as manual stages, because it&rsquo;s smart to check changes before committing. However, I now make it a little easier by following Kaushal&rsquo;s advice to run hugo server with the -p and --navigateToChanged switches. I now keep a pinned tab up in Firefox at localhost:xxxx (choose your own port with -p), and as soon as I export a page, that tab will update to the appropriate page. Fantastic! Makes checking my work much quicker. As for the rest, those steps are all done via 2 git post-commit hooks, one for each repo: in the main repo, I have a post-commit hook that checks if the commit was made on the master branch, and if so, then grabs the commit message & commit hash, then cd&rsquo;s to the website directory & creates a new commit on master with references to the original commit. in the web repo, I have a second post-commit hook, which again checks if we&rsquo;re on master, and if so, generates the website in the public directory (where we have the gh-pages site checked out as a git subtree), cd&rsquo;s there, and commits to the gh-pages brnach. Then both branches are pushed to gh-pages I&rsquo;m in a transitional period where I&rsquo;m moving from self-hosted sites to gh-pages where possible. So I actually regenerate the site in another directory using hugo server -s -d myserver; I also have to update config.toml before doing this to fix a couple of paths. This is all pretty easy to do. Here&rsquo;s the first, somewhat simpler script from the main repo: push_branch=master cur_branch=$(git rev-parse --abbrev-ref HEAD) cur_hash=$(git log -1 --pretty=%h) cur_msg=$(git log -1 --pretty=%B) web_dir=dh-website cur_dir=$PWD # only run if commit is to master if [ &quot;$cur_branch&quot; == &quot;$push_branch&quot; ]; then # if [ &ldquo;$(git status -uno &ndash;porcelain)&rdquo; ]; then # echo &ldquo;uncommitted changes, can&rsquo;t push&rdquo;; # else # cd to website cd $web_dir # commit hcanges echo &ldquo;committing changes to website master branch\n\n&rdquo; git commit -a -m &ldquo;push changes to website from titaniumbones/Digital-History#$cur_hash $cur_msg&rdquo; cd $cur_dir echo &ldquo;pushing master to origin&rdquo; git push -f origin master fi And the slightly more baroque version from the website repo, which pushes to both gh-pages and to hackinghistory (shimano is just an alias for hackinghistory.ca). It’s Time to Finally Take Encryption Seriously!

Table of Contents Encryption Review Messaging Email We have known for a long time that the American intelligence agencies don’t flinch at violating our privacy. Edward Snowden’s revelations have forced some small reforms at the NSA and elsewhere; but those small improvements will doubtless be rolled back, and XKeyscore, PRISM and, ECHELON will be replaced by even more invasive and dangerous technologies. They will be controlled by a government with strong ties to White Nationalists, militias, and various hate groups. In other times and places, governments have circulated death lists of dissenters to paramilitary organizations. I don’t think that’s likely in the USA; but I also think that as citizens, we need to act to protect ourselves from the abuse of power. The only effective tool against government surveillance is end-to-end encryption of communication. Even encryption is likely to be inadequate against a targeted attack by the NSA’s supercomputers and tactical operations; but when it comes to the mass collection of data, properly-practiced encryption protects you from being targeted for more detailed analysis. As a by-product, encrypting your communications makes it a little bit harder (but not much!) for corporations to track your preferences, and also adds a little bit of protection from other malicious agents. The basic building blocks of encrypted communication have been available for a long time, but until recently encryption was clumsy and error-prone. Encryption of email is still somewhat challenging. In the last year, though, two excellent solutions, and one adequate one, have emerged for messaging. I’ll talk about all these in a minute, but first let’s quickly review how encryption works. Encryption Review Almost all contemporary encryption is &ldquo;dual-key&rdquo; &#x2013; each person in a conversation has both a private and a public &ldquo;key&rdquo;. If I want to send you a message, I encrypt it with your public key. After that, even I can&rsquo;t read my message &#x2013; only you can, with your private key, which you don&rsquo;t show to anyone. &lt;p&gt; This sounds simple, but it is actually quite difficult to implement encrypted communication in a simple, easy-to-understand interface. Software developers have struggled with this for a long time. It&amp;rsquo;s worth reading a little more about this: &lt;/p&gt; &lt;ul class=&quot;org-ul&quot;&gt; &lt;li&gt; &lt;a href=&quot;https://ssd.eff.org/&quot;&gt;The EFF&amp;rsquo;s Surveillance Self-Defense Guide&lt;/a&gt; is a comprehensive introduction to encryption practice. You should read it. &lt;/li&gt; &lt;li&gt; &lt;a href=&quot;https://emailselfdefense.fsf.org/en/&quot;&gt;The FSF also has an Email Self-Defense Guide&lt;/a&gt;. Specific to email, but still helpful, and quite a bit shorter. &lt;/li&gt; &lt;li&gt; The EFF used to have a &lt;a href=&quot;https://www.eff.org/secure-messaging-scorecard&quot;&gt;Secure Messaging Scorecard&lt;/a&gt;. The first version has been deprecated, but a new one will be launched soon; read it when it comes out. It will likely supersede much of this information. &lt;/li&gt; &lt;/ul&gt; Messaging The messaging landscape has been changing rapidly. After a long delay, there is now really good encryption available, but some of the interfaces are a little confusing or misleading. &lt;ul class=&quot;org-ul&quot;&gt; &lt;li&gt; &lt;b&gt;&lt;a href=&quot;https://ssd.eff.org/en/module/how-use-signal-android&quot;&gt;Signal&lt;/a&gt;&lt;/b&gt; allows you to send text messages and make phone calls with full, end-to-end, dual-key encryption. You should switch to it immediately. You won&amp;rsquo;t be able to make encrypted phone calls over the regular phone network &amp;#x2013; you will need to use data instead. &lt;/li&gt; &lt;li&gt; As of April, &lt;b&gt;&lt;a href=&quot;https://ssd.eff.org/en/module/how-use-whatsapp-android&quot;&gt;WhatsApp&lt;/a&gt;&lt;/b&gt; now offers dual-key encryption by default, including a key-verification interface. Among the mainstram apps, WhatsApp is probably the best option, but &lt;a href=&quot;https://www.eff.org/deeplinks/2016/10/where-whatsapp-went-wrong-effs-four-biggest-security-concerns&quot;&gt;it&amp;rsquo;s not perfect&lt;/a&gt;, and Signal is better if you can use it. &lt;/li&gt; &lt;li&gt; In June, &lt;b&gt;&lt;a href=&quot;https://www.wired.com/2016/10/facebook-completely-encrypted-messenger-update-now/&quot;&gt;Facebook Messenger&lt;/a&gt;&lt;/b&gt; enabled end-to-end encryption. Encryption is turned off by default and has to be turned on for every conversation, so this is &lt;b&gt;not&lt;/b&gt; as good as WhatsApp, but if your social media life mostly moves through Facebook, you &lt;b&gt;can&lt;/b&gt; enable encryption, and it&amp;rsquo;s worth doing. &lt;/li&gt; &lt;li&gt; Google&amp;rsquo;s new &lt;a href=&quot;https://www.eff.org/deeplinks/2016/09/googles-allo-sends-wrong-message-about-encryption&quot;&gt;Allo&lt;/a&gt; messenging app also permits end-to-end encryption, but it&amp;rsquo;s not on by default, and it&amp;rsquo;s a bit confusing to use. Signal is still ldefinitely preferable. &lt;/li&gt; &lt;/ul&gt; Email Email is tricky. For most people, Thunderbird plus EnigMail is probably the best option, but it is confusing to use and it also &ldquo;leaks&rdquo; information (as will always be the case for email sent through traditional channels). Various projects to fix email were announced in the wake of the Snowden revelations, but the onse I know about have mostly stopped development. It&rsquo;s a hard nut to crack. &lt;p&gt; Still: you should aim to encrypt a relatively high percentage of your emails if you can. It&amp;rsquo;s not good enough to just encrypt a few; the important ones need to be masked by the trivial ones. And encrypting your emails also helps to protect the journalists, activists, ando thers who need to encrypt their communication &lt;i&gt;for their own protection&lt;/i&gt;. We encrypt not just for ourselves, but to preserve essential freedoms for others. &lt;/p&gt; &lt;p&gt; Over the next few months I hope to transition to encryption in most of my messaging; that means that, if you want to be in touch with me online, you&amp;rsquo;ll need to start figuring this stuff out too. Let me know what I can do to help! &lt;/p&gt;

Using the Sikkim Website to keep in touch

Table of Contents Getting Started Location Data Featured Image Showing a post on the &ldquo;Our Journey&rdquo; Page OK, we’re getting down to the wire. This post is for the Sikkim trip students…. I know most of you will keep in touch with friends and family via Facebook &amp; other social media. However, I’m hoping that we will also use the group website as a way to keep in touch while we’re gone. As you know, I’ve set the site up as a major part of our intellectual output for this project, and yo all have accounts that should let you write and publish blog posts &amp; webpages. Here are just a few short tips on how to use the site. I’m hoping those of you who are familiar with WordPress can help the others. Getting Started Use your username and password to get into the site at the admin/login page. To create a new journal entry, go to Posts/New (or tap the + button in the top menu. Compose your message in the editor box, add pictures using the &ldquo;add media&rsquo; button in the editor toolbar, press &rdquo;Publish&ldquo;, and you&rsquo;re done. Location Data Since we&rsquo;re thinking of this as &ldquo;place-based&rdquo; education, location data is pretty important to us. You can add three kinds of location information to your post (underneath the editor box): &lt;dl class=&quot;org-dl&quot;&gt; &lt;dt&gt; GPX Tracks &lt;/dt&gt; &lt;dd&gt; These are recordings of where you&amp;rsquo;ve been (discussed in my last post). If you attach one of these in the &amp;ldquo;GPX File&amp;rdquo; upload box, it will display in your post (but not in the list of all the posts&amp;#x2026;). We&amp;rsquo;ll use these a lot when we&amp;rsquo;re in Sikkim. &lt;/dd&gt; &lt;dt&gt; KML FIles &lt;/dt&gt; &lt;dd&gt; There&amp;rsquo;s a lot of stuff these can do, but the main reason I included them is that in some cases they can be easier to export than gpx tracks. &lt;/dd&gt; &lt;dt&gt; Co-ordinates &lt;/dt&gt; &lt;dd&gt; You can use this to indicate where you are in the moment that you&amp;rsquo;re writing your post. This is what will show up in the &amp;ldquo;archive&amp;rdquo; and &amp;ldquo;our journey&amp;rdquo; pages (yo can look at the test pages for examples of how it will look). &lt;/dd&gt; &lt;/dl&gt; &lt;p&gt; I strongly encourage you to use the geodata options here! It&amp;rsquo;ll help our site out a lot. &lt;/p&gt; Featured Image You&rsquo;ll notice that each post has one special image associated with it, that is used as a packground image in a couple of places. This is the &ldquo;featured image&rdquo;, which you set in a box on the lower right of the writing interface page (or, if you&rsquo;re using your phone, just really far down in the interface). I&rsquo;d really like it if every post had a featured image, so try to choose a good one! Showing a post on the &ldquo;Our Journey&rdquo; Page &ldquo;Our Journey&rdquo; is, as it were, the &ldquo;official&rdquo; record of our trip. Each day one or two of us will write a special post about the day&rsquo;s events; it&rsquo;s expected to be perhaps a little more substantive or public-facing than your regular travel diary. You can move a post into the &ldquo;Our Journey&rdquo; feed by adding the &ldquo;Our Journey&rdquo; category &#x2013; in the &ldquo;Category&rdquo; box. &lt;p&gt; Youu may also notice the &amp;ldquo;Tags&amp;rdquo; box &amp;#x2013; we an talk later about how to make use of tags on the site. &lt;/p&gt; &lt;p&gt; See yo all in India! &lt;/p&gt;

Android Mapping Apps

Table of Contents Offline Map Display Data Storage GPS Tracking Getting your data out REVISED For our trip to Sikkim, I would like students to have a way to map our progress on their phones so that it can be displayed on our website. I’ve spent a bit of time looking at various Android apps (I don’t have an iPhone) and have some thoughts about how to do this; here are my notes, intended mostly for the students but also potentially for other people looking to do something similar. Offline Map Display Most mapping applications rely on a continuous data feed, which will not be available to us in Sikkim. So, what applications offer offline viewing? &lt;p&gt; &lt;a href=&quot;https://play.google.com/store/apps/details?id=com.google.android.apps.maps&amp;hl=en&quot;&gt;Google Maps&lt;/a&gt; allows downloading of areas for offline use. &lt;a href=&quot;https://support.google.com/gmm/answer/6291838?hl=en&quot;&gt;Here are the official instructions for doing so.&lt;/a&gt; The Sikkim area takes about 500mb of space so be aware of that. &lt;/p&gt; &lt;p&gt; &lt;a href=&quot;https://play.google.com/store/apps/details?id=net.osmand&amp;hl=en&quot;&gt;OSMAnd&lt;/a&gt; is an interesting alternative. It is mostly designed for offline use. If you buy the &lt;a href=&quot;https://play.google.com/store/apps/details?id=net.osmand.srtmPlugin.paid&amp;hl=en&quot;&gt;countour Lines Plugin&lt;/a&gt; for $1.99, you can add topographic map contour lines to your map, which is pretty cool. I kind of love it myself, and it&amp;rsquo;s what I&amp;rsquo;ll be using. The interface is perhaps a little less intuitive than the Google Maps interface, but I&amp;rsquo;m getting used to it. You will need to download the world base map, as well as the detailed map for all of India (if you have enough space on your phone) or the information for Sikkim only (if you don&amp;rsquo;t have enough room). The contour map is a separate download, and to be able to use it, you will need to purchase the plugin linked to above. &lt;/p&gt; &lt;p&gt; The free version of the app allows only 7 downloads in total, but we shouldn&amp;rsquo;t need more than that for our trip. I myself bought the &lt;a href=&quot;https://play.google.com/store/apps/details?id=net.osmand.plus&amp;hl=en&quot;&gt;full paid version&lt;/a&gt; but that shouldn&amp;rsquo;t be necessary IIUC. &lt;/p&gt; Data Storage If your phone supports the use of a removable microSD card (most do, mine doesn&rsquo;t), I would highly recommend buying one. You can get an 8gb card for $4.99 at Canada Computers on College St, and that will give you plenty of space for everything you want to put on it. Buy it before you install the app, so that you can choose to store your data on the card rather than the phone&rsquo;s internal memory. GPS Tracking It&rsquo;s nice to have a map on your phone, but most of the time you won&rsquo;t be using the map at all. The real reason for this post is that we&rsquo;d like to place at least one of you on &ldquo;backup tracking&rdquo; duty every day of the trip. For this you need some kind of tracking software, which can generate files in either .gpx or .klm format; and you also need a way to get the data out of the phone and into one of our tablets for writing. &lt;p&gt; There are a number of apps that let you do this. OSMAnd+ (see above) lets you create gpx tracks, though it doesn&amp;rsquo;t let you rename or share them direcly, which is a bit of a pain. Here are some alternatives: &lt;/p&gt; &lt;p&gt; &lt;a href=&quot;https://play.google.com/store/apps/details?id=com.mendhak.gpslogger&quot;&gt;GPS Logger&lt;/a&gt; is a very simple, lightweight app that only records gpx/kml tracks for you. It doesn&amp;rsquo;t display a map or anything. Its main advantage is that it doesn&amp;rsquo;t use much energy compared to other apps. &lt;/p&gt; &lt;p&gt; &lt;a href=&quot;https://play.google.com/store/apps/details?id=com.ilyabogdanovich.geotracker&amp;hl=en&quot;&gt;Geo tracker&lt;/a&gt; is a pretty user-friendly app. Unfortunately it can&amp;rsquo;t make use of OSMAnd&amp;rsquo;s downloaded maps so I&amp;rsquo;m not sur what the display will look like when you don&amp;rsquo;t have any data. &lt;/p&gt; &lt;p&gt; There are &lt;a href=&quot;http://wiki.openstreetmap.org/wiki/Android#Track-making_features&quot;&gt;http://wiki.openstreetmap.org/wiki/Android#Track-making_features&lt;/a&gt;]][zillions of other apps]]. Hopefully one of the above works for you but if not the Open Stree Map wiki is likely your best bet. &lt;/p&gt; Getting your data out REVISED Unfortunately our tablet doesn&rsquo;t support NFC!! So you will need to transfer your files manually onto a USB thumb drive. To do this you will need a file manager. There are lots of them; you can find lists here and here. I don&rsquo;t like any of them all that much but they should work, at least a little. If you have a brand-new version of Android you can also use the built-in file manager. In general it&rsquo;s something of a pain, hopefully we will only be doing that for a few days when we&rsquo;re completely off the grid. &lt;p&gt; &lt;b&gt;NEW:&lt;/b&gt; I had a lot of trouble with most of the file managers, and ended up installing &lt;a href=&quot;https://play.google.com/store/apps/details?id=com.estrongs.android.pop&quot;&gt;ES File Explorer&lt;/a&gt;. Now I&amp;rsquo;m able to move files easily onto an external thumb drive attached to my phone with an On The Go cable (I&amp;rsquo;m bringing 2 so we should be fine on the trip). Note that some users have reported concerns about personal data being stored on Baidu, the Chinese Google. If you have anything on your phone that you don&amp;rsquo;t want the Chinese government to know about, you may want to consider another option. &lt;/p&gt; &lt;p&gt; However: ES File Explorer is &lt;b&gt;MUCH&lt;/b&gt; less painful than other options. With &amp;ldquo;ES Save To&amp;rdquo;, you can save files directly to an external USB drive attached to your phone, which can then be transferred easily to a tablet or compter. &lt;/p&gt; &lt;p&gt; It&amp;rsquo;s still a pain to locate your .gpx files. &lt;/p&gt;
